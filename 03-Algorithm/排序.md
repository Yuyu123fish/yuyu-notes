# lc912.排序数组
https://leetcode.cn/problems/sort-an-array/
给一个数组`nums`，然后在`O(nlog(n))`的时间复杂度下进行排序，思路如下：
1. 在数组中找到一个分割点`int pivotIndex = left + random.nextInt(right - left + 1)`，设分割点的数`nums[pivotIndex]`为分割数，后面遍历的数字都与它进行比较。
2. 然后将`pivotIndex`与`left`进行交换，方便后续排序。
3. 定义`lt=left`表示**小于pivot区间的右边界**，`gt=right+1`表示**大于pivot区间的左边界**，定义变量`i`表示当前遍历的数
4. 然后进行遍历，`while(i < gt)`代表遍历到`gt`就结束了，而不是`right`，因为如果`i==right`时，那么会自动跳过最后一个数字，而且由于`gt`及其右边的数都是严格大于我们的分割数的。
	1. 如果`nums[i] < nums[left]`，也就是比分割数小，那么交换`i`和`lt+1`位置上的数，因为`lt`初始为`left`。然后将`i`和`lt`都+1，那么可以写为`swap(nums, i++, ++lt)`。这里由于我们是从左往右扫过去的，所以`i`左边一定比分割数小，所以可以直接`i++`。
	2. 如果`nums[i] > nums[left]`，那么交换`i`和`gt-1`位置上的数，然后将`gt`--，`i`不减一的原因是，当前交换过来的数，**有可能还是比我们的分割数大**，所以下一轮再比较一次。
	3. 如果`nums[i] == nums[left]`，相等，直接`i++`。
5. 交换我们`left`和`lt`位置的数，因为此时`left`上的数和分割数相等，且`lt`上的数一定也是小于分割数的数。
6. 最后进行两次递归排序`quickSort(nums, left, lt-1)`和`quickSort(nums, gt, right)`。`lt-1`的原因是原本`lt`位置上的数是小于我们的分割数的，在**上一次的交换**中被换成了分割数，所以`lt-1`位置上的数才是小于分割数的数。
```java
class Solution {
    Random random = new Random();
    public int[] sortArray(int[] nums) {
        quickSort(nums, 0, nums.length-1);
        return nums;
    }
    private void quickSort(int[] nums, int left, int right){
        if(left >= right){
            return;
        }
        int pivotIndex = left + random.nextInt(right - left + 1);
        swap(nums, left, pivotIndex);
        int lt = left;
        int gt = right+1;
        int i = left + 1;
        while(i < gt){
            if(nums[i] < nums[left]){
                swap(nums, i++, ++lt);
            }else if(nums[i] > nums[left]){
                swap(nums, i, --gt);
            }else{
                i++;
            }
        }
        swap(nums, left, lt);
        quickSort(nums, left, lt-1);
        quickSort(nums, gt, right);
    }
    private void swap(int[] nums, int left, int right){
        int temp = nums[left];
        nums[left] = nums[right];
        nums[right] = temp;
    }
}
```
- 时间复杂度：
	平均 O(nlogn)，最坏 O(n^2)。由于使用随机 pivot，期望复杂度为 O(nlogn)。
	三路快排在重复元素较多时表现更好，可以减少无意义递归。
- 空间复杂度：
	平均 O(logn)，最坏 O(n)，主要来自递归调用栈。
	排序过程本身是原地排序，额外空间为 O(1)。
