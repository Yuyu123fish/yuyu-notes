# 一.主从复制
## 1.实现原理
- 从库连接到主库时，主库会创建一个`dump`线程，当数据被写入到`binlog`时，`dump`线程把数据推送给从库。
- 从库的`io`线程拿到`binlog`数据后，会把它写入到本地的`relaylog`（中继日志）。
- 从库的`sql`线程读取`relaylog`，重放日志，完成数据同步。
主从复制可以实现读写分离，但需要注意主从一致性问题。

## 2.三种主从复制模式
- 异步复制：mysql默认异步复制，写完binlog就返回，不等从库复制成功，可能导致数据丢失。
- 同步复制：等待所有从库复制完成，缺点是时间慢。
- 半同步复制：
	- 主库提交事务后，至少等待一个从库收到 binlog 并写入 relay log 后再返回。
	- 它比异步更安全，比全同步性能更好，是折中方案。
