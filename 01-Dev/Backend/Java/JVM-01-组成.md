# 一.为什么区分堆和栈
堆和栈是 JVM 中的两个不同区域
+ 堆是存储对象的区域，堆的大小可以根据需要随时调整，堆的管理有垃圾回收器进行，堆内存是多个线程之间共享的。
+ 栈是每个线程独享的一块区域，用于方法调用、局部变量等的存储。
区分的好处
1. 因为他们的**存储内容不同**，可以**分开管理**。堆内存可以用垃圾回收器管理，栈内存可以靠编译器和虚拟机执行完成。
2. 可以做到**不互相影响**。独立开两个不同的区域。堆内存溢出不会影响到栈。栈溢出也不会影响到堆。
3. 可以做到**数据隔离**，因为有了栈，就可以把一些线程独享的局部变量等内容放到栈上，可以做到更好的隔离。而共享的一些数据就可以放到堆上做统一管理。
4. **提升各自性能**。**栈上分配**的效率很高，可以适合分配局部变量等，可以非常的高效。而堆上的内存分配及回收都会相对复杂。这样区分开可以做各自的优化，提升整体效率

# 三.内存结构（运行时数据区）
运行时数据区又分为：方法区，堆，虚拟机栈，本地方法栈，程序计数器
+ 线程私有：程序计数器，虚拟机栈，本地方法栈
+ 线程共享：堆，方法区
1. 程序计数器：记录当前线程正在执行的字节码指令的地址，物理上程序计数器使用“寄存器”完成的。
2. 虚拟机栈
    - 也叫线程栈，每个线程运行时所需要的内存，一个栈由多个栈帧组成。
    - 栈帧对应着每次方法调用时所占有的内存，存储的内容是：方法参数，方法内局部变量，方法返回地址。
    - 一个线程每刻只能有一个活动栈帧，对应当前正在执行的方法。
    - 垃圾回收不涉及 栈内存。
    - 方法递归过多，会导致`StackOverFlowError`栈内存溢出。
3. 本地方法栈
    - 本地方法接口运行时所需要的内存空间。
    - 本地方法是以`native`修饰的方法，用C/C++编写，java有时不能直接和操作系统底层打交道，因此通过本地方法调用操作系统底层功能。
4. 堆
    - 线程共享区域，保存对象实例。
    - **逻辑组成**：年轻代（生命周期短的对象）+老年代（生命周期长的对象）。
    - jdk1.7中堆中有 方法区的实现，即 永久代。
    - jdk1.8把方法区的实现移动到本地内存，叫 元空间。
5. 方法区
    - 线程共享的区域，存储的是类信息，静态变量，常量，编译后的代码，运行时常量池。
    - jdk7方法区的实现叫：永久代，占用的是堆的内存空间，大小固定。
    - jdk8方法区的实现叫：元空间，占用的是本地内存空间，大小自动调整。
6. 直接内存
    - 直接内存不属于JVM管理，而是操作系统的内存。
    - 常见于NIO操作时，用于数据缓冲区。
    - 分配和回收成本较高，但读写性能高，直接内存java代码和系统代码都能访问的到。
## 1.运行时常量池 与 字符串常量池
+ 字符串常量池（堆）
    - 全局共享（每个jvm只有一份）
    - String对象引用，保证字符串不重复
    - 伴随整个jvm过程，由垃圾回收器根据可达性分析回收
+ 运行时常量池（1.7在堆，1.8在本地内存）
    - 每个被加载的类个有一份
        * 类的字面量：数字，字符串的符号引用等
        * 符号引用：类名，方法名
        * 直接引用
    - 随类的加载而创建，随类的卸载而销毁
