# 一.字符串
## 1.String，StringBuilder，StringBuffer有什么区别？
+ `String`是不可变字符序列，每次修改都会new一个对象，线程安全
+ `StringBuilder`是可变的，线程不安全，但效率高
+ `StringBuffer`是可变的，线程安全，使用`synchronized`修饰，但效率低

# 二.方法
## 1.为什么重写equals()要重写hashcode()？
为了确保对象在哈希集合中正常工作，因为需要hashcode来分配存储桶的位置。这些集合在寻找对象位置时，**会先通过`hashcode`来找到位置**，**再通过`equals`来精确比较**。如果不重写，可能两个逻辑相同的对象有不同的哈希值，导致哈希集合存储重复元素，无法正常查找对象。

# 三.反射
## 1.反射基础
Java 反射是指程序在**运行期间动态获取类的信息**，并**动态创建对象、调用方法、访问属性的机制**。
它的核心入口是 `Class` 对象，常用的类有 `Class`、`Method`、`Field`、`Constructor`。
- 反射常用于框架开发，例如动态代理，或者比如 Spring 创建 Bean、依赖注入、读取注解、AOP 等。可以让代码更灵活。
- 缺点是**性能相对较低，破坏封装，无视泛型参数，并且很多错误只能在运行时暴露**。

## 2.反射的操作
1. 获取**类的Class对象**：知道具体类，类的全路径，对象实例，类加载器。
2. 创建对应的**对象实例**。
3. 通过方法签名等信息可以**获取到方法**。
4. 还可以对**类的参数进行修改**。

## 3.AOP如何利用反射？
AOP 主要在两个阶段用到反射。
- **第一是在容器启动时**，Spring 通过**反射扫描 Bean** 的类信息、方法信息和注解信息，判断哪些方法**匹配切点**，需要被增强。
- **第二是在运行时**，Spring 会为目标对象创建代理对象，当调用目标方法时先进入代理逻辑，执行前置、后置、异常等通知，再调用真实方法。
	- 如果是 **JDK 动态代理**，调用真实方法时会通过 `Method.invoke` **反射执行目标方法**；如果是 CGLIB，则主要通过动态生成子类的方式进行方法增强。
