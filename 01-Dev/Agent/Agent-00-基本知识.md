# 一，Agent的几种执行策略
- ReAct：边思考边行动，适合中等复杂度的任务。
	- Reasoning + Acting：思考+行动。
	- 每一步都重新推理，灵活但效率低。
- Plan-Execute：先制定计划再整体执行，适合复杂的多步骤任务（比如数据分析）。
	- 一次规划全部执行步骤，效率高但不灵活
	- 所以可以使用**Plan-Execute**先分析步骤，再**ReAct**执行每一步。
- Reflection：执行后自我反思，适合需要迭代的任务（比如代码审查）。
- LLMCompiler：分析后并行执行，适合可以并行执行的任务。
- Multi-Agent：多智能体协作。

# 二，记忆
## 1.短期记忆
每次与大模型进行对话，都是将全部上下文发送给大模型，等待大模型生成答案，但是大模型有上下文窗口上限，随着文本量的增大，不可能将**全部上下文**都发送给大模型，因此需要对上下文进行处理，由此引入了短期记忆。

对于短期记忆的处理方式有以下几种：
- 直接截断：直接截断前面的40%的上下文，好处是时间快，坏处是上下文丢失较多，对于大模型理解背景和生成答案影响较大。
- 滑动窗口：达到一定阈值（比如80%）后，采取将其中的50%（或者其他）进行总结摘要，然后替换到原来的上下文，好处是大模型不会丢失很多关键信息，坏处是会额外耗费时间和token。
	- 实际开发中，可以使用**异步总结摘要**的方式进行处理，比如一次对话结束后，先判断是否需要进行总结摘要，如果需要就异步执行总结摘要过程然后替换上下文。

## 2.长期记忆
长期记忆支持跨对话存储和检索，更有利于大模型提取长期存在的事实。

# 三.工具
工具赋予了智能体额外的能力，比如智能体无法准确得知未来的天气预报是什么，以及无法获取一些定位信息，那么我们可以为智能体提供我们定义好的工具来扩展智能体的能力。

定义工具：
- 描述清晰：描述清楚工具是做什么的，得到什么结果。
- 职责单一：一个工具只做一件事。
- 错误可读：工具调用发生错误时，要给大模型清晰的错误结果。
- 参数限定：使用枚举等方式来减少大模型的幻觉。
