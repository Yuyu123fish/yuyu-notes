# 二.长期记忆
## 1.长期记忆存了什么？如何界定？
- 长期记忆主要存**用户维度、长期稳定、对后续回答有帮助的信息**。比如用户的背景与偏好，正在坚持的事情，或者用户明确的要记住的事情。
- 不应该存**临时问题，工具调用问题，系统提示词或与用户无关的内容**。
- 记忆抽取使用温度为0的禁用工具的模型，对话后异步抽取，使用提示词和白名单黑名单约束。
	- 白名单：用户背景，长期偏好，长期目标等；
	- 黑名单：系统提示词，智能体身份，一次性问题，工具调用结果等；

## 2.了解Mem0吗？
它不是大模型也不是数据库，而是帮智能体做”记忆抽取，存储，更新和召回“的框架/平台，是AI Agent的长期记忆基础设施。可以让大模型应用跨会话保存上下文。
1. 从对话了里**抽取**值得记住的信息。
2. 把记忆**保存**起来。
3. 下次对话按当前问题**召回**相关记忆。
4. 持续**更新**记忆。

## 1.同一个对话问了A问题和B问题怎么保证不乱注入？
每一轮对话做独立召回，走 文本+向量 的双路召回，而不是随便把长期记忆和知识库内容塞进去。
- 文本检索依靠（多查询扩展后的）倒排索引，关键词可以精确命中。
- 向量检索依靠语义相似度。
双路召回之后做排序，然后去重，按照Top8的文本数据注入。
它本身是**基于问题的驱动**的召回，如果A和B差异较大召回内容是几乎无关的。如果相似，那召回有共同的是合理的。

## 2.TopK内容也可能不相关怎么解决？
TopK只能降低乱注入的概率，不能绝对保证。可以加几层过滤：
- 第一层是**相似度阈值**，低于阈值的不注入。
- 第二层是**来源隔离**，长期记忆，私有知识库和公共知识库分开标注。
- 第三层是注入带上**来源信息**，让模型知道信息来自哪里。
- 第四层是对长期记忆写入做**黑名单和白名单**处理，避免脏数据进入数据库。
